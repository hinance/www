hinance: automated personal bookkeeping for hackers



Install on Arch Linux Source & Docs



Motivation

I've been doing bookkeeping in my family for a few years using a variety of tools. Unfortunatelly, neither of them had a combination of features I wanted, which is: showing actual vs. planned expenses by categories; automated data import from banks and shops websites which don't have API; integration of data from shops and banks (for example, I want to see that $100 from credit card was paid for an online order, which consists of $10 T-shirt, $30 video game, $50 microwave oven, $5 shipping expenses and $5 taxes). I resorted to writing such a tool myself, and now I'm happy to share it with you all.

Features

It runs continuously as a daemon, scrapes the data from banks and shops websites, transforms it into single consistent bookkeeping journal and generates the report.

User adjusts transformation of the scraped data into the bookkeeping journal by writing a few callbacks in Haskell. Here's how Phillip and Leela's report looks like without custom adjustmentswithout custom adjustments. And here's the same report with custom adjustmentswith custom adjustments.

Uses Weboob for scraping. It's open-source: you can create new modules for the websites you're using, or fix existing ones when websites get updated.

Integrates the data from shops and banks websites into a single consistent bookkeeping journal. For example, here's how transactions group generated from online order looks likelooks like in the report.

ShowsShows history of actual vs. planned transactions split by categories. Individual ordered items, as well as bank transactions, can be categorized.

Report comes in two flavors: desktop and mobile.
